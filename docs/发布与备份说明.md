# 发布与备份说明

## 公开仓库范围

公开仓库只保存可复用源码、公开文档、测试与预览资源。以下内容必须留在本机可信存储，不进入 Git 历史或 GitHub Release：

- `docs/私密/` 与 `backups/`。
- 设备镜像、MTD/eMMC 转储、配置归档、DPAPI 文件和离线安装包。
- 管理/Wi-Fi 密码、代理订阅、节点、密钥、令牌和本机密码辅助脚本。
- `.tmp/`、`tmp/`、缓存、测试临时文件及历史 `.bak`。

## Release 备份内容

每个 Release 附带一个从已提交 Git 树生成的源码压缩包和对应 SHA-256 文件。使用 Git 树作为来源，确保备份资产与标签内容一致，且自动遵守提交前的排除与敏感信息审计结果。

Release 说明按“新增、修改、修复、移除或不再发布、安全、验证”列出变化。不要把运行中的地址、密码、订阅或私密备份路径写进发布说明。

## 恢复方式

下载 Release 资产并核对 SHA-256，再解压到新的工作目录。Release 资产是源码与公开文档备份，不是路由器整机镜像，也不能替代兼容固件、设备配置归档或外置数据备份。
