## 结论

`192.168.0.242` 的主要残留位于内存文件系统 `/tmp`，不是持久化 overlay。清理旧备份展开目录、已离线保存的重复归档、OpenClash 安装包和一次性诊断文件后，`/tmp` 从 `624.9 MiB / 63%` 降到 `67.3 MiB / 7%`，释放约 `557.6 MiB`。

## 可复用脚本

使用 [`cleanup-approved-temp.sh`](../scripts/router/cleanup-approved-temp.sh)。脚本默认 dry-run，只接受操作者显式传入的规范 `/tmp/*` 绝对路径；传入其他目录或 `/tmp` 本身会以 `REFUSE_OUTSIDE_TMP` 退出。脚本通过 `readlink -f` 复核解析后的路径仍在 `/tmp` 内，并拒绝现存或缺失的含 `..`、多余斜杠及符号链接等非规范路径。只有规范但不存在的目标才输出 `SKIP_MISSING`。只有添加 `--apply` 才删除。

先备份并逐项确认路径不在使用，再执行：

```sh
sh ./cleanup-approved-temp.sh \
  /tmp/example-old-backup \
  /tmp/example-old-package.ipk

sh ./cleanup-approved-temp.sh --apply \
  /tmp/example-old-backup \
  /tmp/example-old-package.ipk
```

每台设备独立生成候选清单。不要把本机日期化路径直接复制给其他设备，也不要增加自动扫描后删除、开机自动清理或定时清理任务；显式白名单能避免误删正在使用的恢复点。

## 失败处理

- 出现 `REFUSE_OUTSIDE_TMP`、`REFUSE_RESOLVED_OUTSIDE_TMP`、`REFUSE_NONCANONICAL` 或 `PATH_RESOLUTION_FAILED` 时，修正候选清单，不绕过路径保护。
- dry-run 出现意外路径或候选仍被进程打开时停止，不执行 `--apply`。
- 删除过程中出现 `DELETE_FAILED` 时停止后续清理，保留现场并重新盘点。
- 服务、网络或关键配置哈希异常时停止写入，使用本次改前备份恢复对应文件，再重复完整黑盒验收。
- `/tmp` 是内存文件系统，重启本来就会清空；不要为了恢复无业务价值的旧临时副本而覆盖当前配置。
