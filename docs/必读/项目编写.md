# #CODE — 收尾判据 + 工作范式

## 触发与收尾

写完代码、改完文档、跑完实验、调完命令或声明完成前，主线主动自查：

| 项 | 要求 |
|---|---|
| `claim` | 说清完成了什么 |
| `evidence` | 给出命令输出、测试通过或文件落定的真实证据；禁止编造 |
| `necessity_recap` | 本会话实际 `dd get` 过的篇目各写 1 行主旨；抄 frontmatter description 不算，未拉取的不列 |
| `missing` | 列出尚缺的验证 |
| `status` | 仅 `done_with_evidence` 通过；`needs_check` / `candidate` 未通过，继续补证据 |

跳过或伪造证据时任务不作数，用户可终止会话。只检查主线这个“会话”单位，派出的 subagent 免检。

## Claim → Evidence

按 claim 类型选 evidence，禁止替代：

| Claim | 必需 evidence |
|---|---|
| 机制 / 模式 / 算法可用 | 在真任务运行，并用输出 log 对比主线自读 |
| 文档描述清楚 | Understand-check：N ≥ 2 agents，一致度 ≥ 90 |
| 事实完整保留 | Harness audit：`retention_pct ≥ 95` |
| 代码正确 | build / compile 通过 + 手动 smoke test |
| 交付可用 | 端到端 demo + 真实输出 |
| 引导 agent 行为的 doc 改动触发力可用 | 用户原 prompt 一字不加，派黑盒 subagent 复现并比较改前/改后行为；3 段对齐只验改动合规，不验触发力 |

不得混用：Understand-check ≠ 机制验证；build 通过 ≠ 交付可用；audit retention ≠ 语义清晰。

## 不可替代性与持久化

用户请求依赖仅存于本会话的知识时立即执行：

1. 检查下个 Agent 冷启动读完所有 doc 后仍缺什么：隐式规矩、口头修正、失败 lesson、本会话新决策。
2. 检查 claim 的机制、模式或结论是否跑过真任务，而非只做描述质量 check。
3. 把 review 结果交给用户，询问写 doc、丢弃或延迟；跳过即制造 lock-in。

主线学到新知识或产出实质内容后，必须询问是否持久化。用户选择延迟时不主动写，并在收尾 evidence 中列延迟项数量；持久化路径运行 `dd get MEM` 获取。

## 通用设计原则

- 列举 / 清单命令（“list all X”）直接读数据源，禁止静态扫源码或硬编码列表；新增一项需改 ≥2 处说明模式错误。
- 等状态时用事件监听或短命令 lazy scan；禁止 `while (1) { sleep; check }`。短命令可在每次入口扫描一次，不长驻。
- 多 AI 并行时运行 `dd get TEAM`。
- doc 是跨 CLI 可读的唯一 canonical sink；Agent 只执行，不充当知识库，替换主线不得留下 debt。

## 改动后的必做流程

新建、修改或删除项目命令 / 核心函数，修改数据结构（持久化 schema 涉及兼容性风险），或修改生命周期、状态转换、关键路径等核心行为后，执行两步。

### 1. 判断是否写 skill

skill 只承载命令 help 装不下的内容：

| 类型 | 判例 |
|---|---|
| W1 多步工作流 | 命令 A→B→C 的顺序或中间条件无法由 help 讲完 |
| W2 跨命令坑 | 本命令完成后必须调用另一个命令 |
| W3 底层 spec | 产物文件字段语义 |
| W4 数值经验 / 视觉判据 | 阈值或“人眼截图核验” |
| W5 参考实现 | 实际使用需 20+ 行 orchestration |
| W6 环境 / OS 依赖 | 特定平台限制 |

| 命中数 N | 处置 |
|---:|---|
| 0 | 不写 |
| 1 | 写入 stderr / help 自述 |
| ≥2 | 写入项目实际采用的 canonical skill 文档位置；目录或单个 Markdown 由项目指定，并通过 `DD_DOCS` / `--docs` 提供给 dd |

拿不准时，假设删掉全部 skill、只留命令 help：下个 agent 能冷启动完成则不写。

### 2. 验证收敛

运行 `dd get TEST`，派黑盒无上下文 subagent 二次对齐。

## 自述式命令

把命令 help + stderr 作为参数、`error_kind`、输出 shape 的唯一真相源，doc 禁止重列。命令不好用就补 stderr，不用 doc 补；静默命令视为 bug。
