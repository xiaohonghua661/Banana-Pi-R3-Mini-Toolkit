# #DOC — 文档编写

## 流程

先判断是否需要写文档、是否产生外部思考、是否开发额外内容，再执行：

1. 总结拟写 doc。
2. 征得用户同意。
3. 按三分法则编写。
4. 运行 `dd get TEST`，派黑盒 subagent 二次对齐。
5. 运行 `dd get REDOC` 的通用收敛流程；一致度 ≥ 90 才收敛。

## 先确定真相源

命令行为以代码中的命令定义（含 `error_kind` handling）为唯一真相源。skill 只保留代码装不下的 workflow、非显然 why 和跨命令坑；此时“每篇自足”指 skill 正文与命令 help 合起来自足，因为 agent 使用命令时必然看到 help。命令不好用时补 `error_kind` / stderr 自述，不用 doc 补。

仅当 doc 本身是唯一真相源时，must-have 中的完整失败节和真实输出示例才强制适用。

## 三分法则

| 分类 | 要求 |
|---|---|
| Must-have | 失败节同时包含 `error_kind` 名、handling、失败输出示例；在 params 表的 `error_kind` 列或失败节至少暴露一次名字；成功与失败输出都必须真实、字段完整、无“略”和占位；事实本地写全；用 imperative form（“调用 X”，不用“我们可以调用 X”）；仅为非显然步骤写一句 why；只写已运行验证的内容 |
| Optional | 参数用表或散文；散文须含类型、默认值、required；shell 示例可用真实参数或占位；frontmatter 结构、标题措辞、维护规则节均近乎无差 |
| Harmful | 用跨 doc 引用替代本地事实最严重；输出示例用“略”或占位；缺失败节；emoji + 散文 + 无表叠加会放大类型推断幻觉 |

## Frontmatter

每篇 dd tag skill 必须包含：

```yaml
---
name: <TAG-NAME>          # 唯一，dd 按此索引，惯例全大写
description: 一两句触发+目的  # ≤300 字，供 dd search 打分
---
```

禁止 `related:`、`triggers:`、`category:`，避免跨引用并保持 flat frontmatter。

## 跨文档引用

禁止用下列引用替代当前 doc 的必需事实：

- 硬编码文件名，如 `见 XX.md`、`按 XX 规约`。
- 抽象指路，如 `见 xxx 目录`、`按 xxx 口吻`、`用 dd search "<关键词>" 命中`。
- 用 `related:` 列其他 skill。

允许两种情况：

- `dd get <TAG>` 是可执行动作，不是跳转依赖，读者自行决定是否拉取。
- 项目冷启动入口要求 Read 某目录全部文档，这是 inventory；仍不得用目录指路替代当前 doc 必需事实。

跳转浪费 token、割裂上下文，重命名不同步会牵一发动全身。把必需的 1–3 行原句写入当前 doc；少量重复优于 coupling。

## 消化用户口头规则

- 改成 imperative form，删除“你要”“我觉得”。
- 仅为非显然内容补一句 why。
- 删除情绪词和上下文假设。
- 压成无寒暄、无叙述的 skill 口吻。

用户表达依赖上下文，直接照抄会让下游 Agent 阅读突兀。

## dd 索引

始终扫描 `<INSTALL_DIR>/docs/**.md`。项目文档位置不设固定名称：由项目或 agent 用 `DD_DOCS=<path-list>` 或命令参数 `--docs <path-list>` 指定一个或多个目录、或单个 Markdown；dd 递归扫描目录并按 frontmatter `name` 建索引，后指定路径覆盖前面的同名条目。无缓存，编辑实际文档后，下次命令重新扫描。

| 意图 | 命令 |
|---|---|
| 全览 | `dd list [--json] [--docs <path-list>]` |
| 搜索 | `dd search <query> [--docs <path-list>]`：name +3、description +1，按得分排序 |
| 读取 | `dd get <name> [--docs <path-list>]`：输出含 frontmatter 的完整 Markdown |

Agent 按 `dd list` → `dd search` → `dd get` 使用文档；禁止 blind Read，否则会漏 index。

## 失败

| 现象 | 处置 |
|---|---|
| `skill_not_found: <name>` | 名字错误；先运行 `dd list` |
| frontmatter 已加但 `dd list` 不显示 | 检查两条 `---` 是否各占一行，以及是否缺 `name` 或 `description` |
| 中文文件名在某些 shell 报错 | 给路径加引号 |
