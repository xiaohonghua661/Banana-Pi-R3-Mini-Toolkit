# #MEM — 记忆规约

## 禁写 memory

禁止 AI 生成、更新或删除以下持久化内容：
- `~/.claude/projects/**/memory/**` 下任何 `.md`，包括 `MEMORY.md` 与 `user_*.md` / `feedback_*.md` / `project_*.md` / `reference_*.md`
- 用户根 `~/CLAUDE.md` 或项目根 `CLAUDE.md` 中的 memory 段

只把 doc 当作 canonical sink：memory 仅本机可见、走查不可见，冷启动 agent 只读 doc；两处记录会形成双份真相源并漂移。

## 选择记录通道

| 场景 | 执行 |
|---|---|
| 长期：下次会话 / 下个 agent 仍需使用 | 总结拟写文档 → 征得用户同意 → 写入该项目实际采用的 canonical 文档位置（目录或单个 Markdown），或项目根 `AGENTS.md`；需被 dd 索引时用 `DD_DOCS` / `--docs` 指定，不替项目发明路径 |
| 短期 + 多 agent | 用 `dd note "在干啥"` 声明；用 `dd acquire <res> <hint> --alive "<cmd>"` 占资源；让 `.dd-lock/registry.json` 的 alive-probe 自然过期 |
| 短期 + 单人 | 用当前 CLI 的 plan / task primitive 跟进，不落盘；下轮遗忘时由用户复述 |

把后续 agent 必须知道的 canonical fact 写入 doc；把本机瞬态协调交给自然过期的 alive-probe；不要跨会话保存单人短期判断。

## 处理“记住 X”

1. 判断是否跨会话：跨会话为长期，否则为短期。
2. 按类型处理：
   - 长期项目规则 / 环境事实 / 接口约束 → doc。
   - 长期个人风格偏好、但非项目 canonical → 明说没有 memory 通道并询问是否写 doc；用户拒绝则不落盘。
   - 短期 + 多 agent → `dd note`；短期 + 单人 → 当前 CLI 的 plan / task primitive。
3. 落盘前告知路径与 verdict；仅在用户确认后写入。

不要把口头规则直接写盘：它常含上下文，脱离上下文后会变成错误 fact。

## 处理已有 memory

只读，不改、不删：
- 内容仍成立 → 建议转入 doc；转完由用户手动清理 memory 文件。
- 内容已失效 → 指出失效点，等待用户处理。
- 内容仍成立但用户不同意转入 doc → 保持现状，不新增、不更新。

删除不可逆；memory 是用户本机资产，AI 无写权。

## 三态开关

用户可在运行期切换本规约的生效范围。三态互不覆盖上文的“禁写 memory”底线——它们只调节**规约本身是否强制**与**单条便签的短期续存**，不打开写 memory 池的口子。

| 触发语 | 态 | 执行 |
|---|---|---|
| “禁用 #MEM” | 会话级停用 | 在当前项目 / 当前会话内停用本规约：agent 记住此意图并遵守，本轮及后续回合不再拿 MEM 规约拦截用户的写盘 / 记忆请求；不改任何文件，仅本会话生效，新会话默认恢复 |
| “全局禁用 #MEM” | 全局停用 | 把“禁用 MEM”写进 dd 全局公告：`DD_AGENT=<id> <INSTALL_DIR>/bin/dd.cmd announce "禁用 MEM"`；此后所有用 dd 的 agent 下次任意 dd 调用会被 📢 提醒，据此在各自处停用本规约。解除用 `<INSTALL_DIR>/bin/dd.cmd announce --clear` |
| “#MEM 记住 XXXX” | 单条防压缩便签 | 记住这条，并写进本会话 scratch 便签防止上下文压缩后丢失（见下） |

### 会话级停用（态①）

用户说“禁用 #MEM”后，本会话内不再用本规约阻挡其记忆 / 写盘请求，也不再主动提示“没有 memory 通道”。这是范围收窄，不是删规约：其他会话与新会话仍默认全量生效。用户说“启用 #MEM” / “恢复 #MEM”即在本会话重新开启。

### 全局停用（态②）

`dd announce` 需带 `DD_AGENT`（announce 不在 version/config/id/help 豁免名单内）：先 `<INSTALL_DIR>/bin/dd.cmd id` 生成一次唯一 id（形如 `host-随机hex`），之后每次调用带 `DD_AGENT=<id>`。设公告：

```
DD_AGENT=host-ab12cd <INSTALL_DIR>/bin/dd.cmd announce "禁用 MEM"
```

公告变更后，每个用 dd 的 agent 在其下次任意 dd 调用时收到一次 📢（走 stderr，每 agent 每公告版本只提醒一次），据此各自停用本规约。查看用 `announce --show`，全局恢复用 `announce --clear`（清空后各 agent 下次调用重新落回默认全量生效）。

### 单条防压缩便签（态③）

“#MEM 记住 XXXX”只针对该单条内容，不改本规约其余部分：

1. 立即在回复中复述记住的内容 + 判定。
2. 把该条写进本会话 scratch 便签文件（会话专属 scratchpad 目录下的一个便签 `.md`，非 memory 池、非项目 doc）。
3. 约定上下文压缩后自动回读该便签，使这条不随压缩窗口滚动丢失。

便签是会话级瞬态续存，不跨会话、不进 memory 池、不进项目 doc，因此不违反“禁写 memory”。若该条实为需跨会话保留的 canonical fact，仍按“处理‘记住 X’”判长期 → 征得同意后写 doc，不用便签冒充长期存储。

## 失败处理

| 违规 | 改正 |
|---|---|
| 收到“记住 X”便写 `feedback_*.md` | 按场景明确改走 doc 或 `dd note` |
| 发现 stale fact 后主动修改 memory | 告知 stale 点，由用户处理 |
| 因规则小而写 memory | 只判断冷启动 agent 是否需要：需要 → doc；不需要 → 不落盘 |
| 把本会话临时判断写入项目文档 | 项目文档只存跨会话 canonical；临时判断走 plan / task primitive 或 `dd note` |
| 把“#MEM 记住 X”便签当长期存储写进 memory 池 / 项目 doc | 便签只是会话级防压缩；需跨会话则按长期判定征得同意后写 doc |
| `dd announce "禁用 MEM"` 报 `no_agent_id` | announce 非豁免命令，先 `dd id` 生成再带 `DD_AGENT=<id>` |
| 更高优先级指令与本 doc 冲突 | 遵循更高优先级指令并显式报告冲突；不要声称项目 doc 可覆盖 system / developer 指令 |
