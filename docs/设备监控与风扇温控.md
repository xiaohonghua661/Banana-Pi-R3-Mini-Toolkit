# 设备监控与风扇温控

## 当前结论

- LuCI 外层界面已使用中文。
- 当前 Netdata 版本为 v1.38.1。该旧版仪表盘没有官方简体中文界面或可在设置中启用的中文语言包。
- 浏览器整页翻译只能改变当前客户端的显示，不会永久修改路由器上的 Netdata。
- BPI-R3 Mini 的 PWM 风扇已由 `r3mini-fan` 服务接管。服务按温度区间和回差调整档位，避免风扇在临界温度反复启停。

## Netdata 中文显示

保留 Netdata 原始仪表盘，不安装来源不明的中文包，也不为中文界面升级监控服务。v1.38.1 的官方 GUI 源码没有 `locales`、`i18n` 或翻译资源；强行维护自定义汉化前端会增加升级和故障恢复成本。

需要临时中文显示时，在电脑浏览器中使用“翻译成中文”。图表名称和动态文本可能无法全部翻译。

## 风扇硬件接口

当前设备实际暴露以下接口：

| 用途 | 路径 | 已核验状态 |
|---|---|---|
| CPU 温度 | `/sys/class/thermal/thermal_zone0/temp` | 可读，单位为毫摄氏度 |
| 内核温控模式 | `/sys/class/thermal/thermal_zone0/mode` | 服务运行时为 `disabled` |
| 原始 cooling state | `/sys/class/thermal/cooling_device0/cur_state` | 仅保留诊断；当前服务不再依赖该值调速 |
| PWM 控制器 | `/sys/class/hwmon/hwmon1/pwm1` | 反相 PWM：`0` 为满速，`255` 为停转 |
| PWM 控制开关 | `/sys/class/hwmon/hwmon1/pwm1_enable` | 实测为 `1` |

`hwmon1` 实际指向 `/sys/devices/platform/pwm-fan/hwmon/hwmon1`。不要使用旧启动脚本注释中的 `hwmon2/pwm1`；当前 `hwmon2` 属于 Wi-Fi 设备，不是风扇。

BPI-R3 Mini 设备树定义的 cooling levels 为 `<255 128 80 0>`。2026-07-18 实机验收发现该风扇是反相 PWM；原先把 `255` 当成满速会导致高温时风扇停转。修正后的 `r3mini-fan` 直接控制 PWM，不再通过 `cur_state` 推断档位：

| 逻辑档位 | 用途 | `pwm1` |
|---:|---|---:|
| `0` | 停转 | `255` |
| `1` | 低速 | `128` |
| `2` | 中速 | `80` |
| `3` | 高速 | `0` |

参考来源：[OpenWrt BPI-R3 Mini 设备树](https://github.com/openwrt/openwrt/blob/main/target/linux/mediatek/dts/mt7986a-bananapi-bpi-r3-mini.dts)。

## 当前温控曲线

配置文件为 `/etc/config/r3mini_fan`，温度单位为毫摄氏度。阈值可在 LuCI 页面调整。2026-07-19 RST 恢复后，用户确认默认曲线以 40°C 为高速升档点，继续执行“尽量压在 40°C 以下”的策略。

| 动作 | 升档配置与温度 | 降档配置与温度 | 回差 |
|---|---:|---:|---:|
| 停转 ↔ 低速 | `start_temp=35000`（35°C） | `stop_temp=30000`（30°C） | 5°C |
| 低速 ↔ 中速 | `medium_up=38000`（38°C） | `medium_down=35000`（35°C） | 3°C |
| 中速 ↔ 高速 | `high_up=40000`（40°C） | `high_down=37000`（37°C） | 3°C |

`interval=5`，即服务每 5 秒读取一次 CPU 温度。达到升档温度后提高风扇档位；温度必须降到对应降档温度以下才降低档位。已部署脚本设计为在传感器读数无效时切到高速档，但尚未通过人为制造传感器故障进行实机验证。

已部署脚本按以下关系校验配置；当前配置已正常运行，但尚未专门执行错误配置的负向测试：

```text
stop_temp < start_temp <= medium_up < high_up
medium_down < medium_up
high_down < high_up
```

## 调整方法

1. 打开 LuCI“系统 → 风扇管理”。
2. 调整六项温度；页面会校验范围与阈值顺序。
3. 点击“应用并重启服务”。
4. 确认运行策略为 `AUTO`；高温高速时 PWM 应显示 `0`。

优先扩大升档温度与降档温度之间的差值，减少频繁切换，但不要把高速升档设得高于实际温控目标。默认 `high_up=40000`；如果为了安静手动提高该值，就同时放宽了“低于 40°C”的目标。若风扇输出 100% 时仍无法降到目标以下，检查风扇、散热片、导热材料和机箱风道。

## 服务与文件

| 路径 | 用途 |
|---|---|
| `/etc/config/r3mini_fan` | 可调整的温控参数 |
| `/usr/sbin/r3mini-fanctl` | 温控循环程序 |
| `/usr/sbin/r3mini-fan-web` | LuCI 使用的受限状态与控制接口 |
| `/etc/init.d/r3mini-fan` | procd 服务入口 |
| `/etc/rc.d/S99r3mini-fan` | 开机自启动链接 |
| `/www/luci-static/resources/view/system/r3mini-fan.js` | LuCI“风扇管理”页面 |
| `/usr/share/luci/menu.d/luci-app-r3mini-fan.json` | 菜单入口 |
| `/usr/share/rpcd/acl.d/luci-app-r3mini-fan.json` | 仅允许登录用户执行受限 helper |

服务运行时已实测将内核温控模式切换为 `disabled`，避免内核 governor 与用户态服务同时写入同一风扇档位。脚本设计为停止服务或正常退出时写回 `enabled`；目前尚未完成停用后的内核调速联动测试，不能据此断言内核自动温控已经可用。

## 验证记录

2026-07-19 以 `backup-immortalwrt-2026-07-18.tar.gz` 为 RST 后新基线重新部署。备份不含风扇的 7 个运行文件，本轮已补齐并启用开机启动。用户截图和实机回读确认默认阈值为 `30/35/35/38/37/40°C`；等待一个 5 秒采样周期后，约 48.3°C 时状态为 `AUTO / 高速 / pwm=0 / mode=disabled / running=true`。手动低、中、高三档分别回读 `128/80/0`。LuCI 路由和视图资源均返回 HTTP 200；浏览器自动化两次加载超时，页面视觉状态以用户提供的实机截图为证，未宣称自动化点击通过。

2026-07-18 首轮验收曾把 `state=0 / pwm=255` 误判为高速；用户现场确认风扇没有转动。该结论已作废，原因是硬件使用反相 PWM。

直接写入 `pwm=0` 后，温度由约 46.4°C 回落到约 44.7°C；设备没有 `fan1_input` 转速传感器，因此软件只能核对 PWM 和温度趋势，物理旋转仍需现场目视或听音确认。

## 故障处理与恢复

### 服务未启动

检查 `/etc/config/r3mini_fan` 中的数值和大小关系。脚本设计为在配置含非数字或阈值关系错误时记录错误并退出；该失败路径尚未执行负向测试。

### 温度读数失效

脚本设计为将 `pwm1` 写为 `0`，使反相 PWM 风扇进入高速档。该保护路径尚未人为触发验证；若遇到实际读数故障，先确认网页显示“高速 / PWM 0”，再修复传感器路径并重启服务。

### 恢复内核自动温控

在 LuCI“系统 → 启动项”中停止并禁用 `r3mini-fan`。停止后读取 `/sys/class/thermal/thermal_zone0/mode`，确认是否变为 `enabled`；随后观察温度变化时 `cur_state` 与 `pwm1` 是否联动。两项都验证通过后，才能把内核自动温控视为可用。当前尚无这组实测记录。

### 无法保持目标温度以下

先确认网页显示“高速”、`pwm1=0` 且能看到或听到风扇旋转。三项均成立时，控制输出已经满速；检查物理散热，不要继续降低阈值或同时启动其他风扇控制脚本。
